## 4. ★ THE DEFINING FEATURE — tau3 is a 3-LLM benchmark (relay-heavy)
Three independent model-call paths (all via the relay), NOT one:
1. **Assistant/agent LLM** — `tau2 run --agent-llm` / Harbor `tau3_llm_agent` / `run_tau3_llm_agent.py` → **gpt-5.4-mini xhigh** (our model).
2. **Simulated-user LLM** — the `tau3-runtime` MCP sidecar (`environment/runtime-server/server.py`) builds user-sim args from `OPENAI_API_KEY`/`OPENAI_BASE_URL`/`TAU2_USER_*` → **a second LLM per turn**.
3. **NL-assertion evaluator LLM** — verifier `tests/evaluate.py` via tau2 evaluator, `TAU2_NL_ASSERTIONS_MODEL` → **a judge LLM at scoring**.
**Implication:** relay load ≈ (agent turns + user turns + judge calls) × 375 tasks — far heavier than TB2.1 (single agent). Relay-budget planning + concurrency must account for 3× call classes. Choose the user-sim + judge models deliberately (per [[feedback_skillforge_judge_openai_only]] / [[feedback_prefer_openai_closed_source]] — likely a gpt-5.x for user-sim + judge; **not** Opus quota).

## 7. Oracle-gold line design — ★ NOT 0-token (key difference from TB2.1)
TB2.1 oracle = apply reference solution = **0 model tokens**. tau3 oracle applies the reference agent actions, **but the simulated-user LLM still drives the dialogue and the NL-judge still scores** → oracle-gold **consumes user-sim + judge tokens** (only the agent side is free). Design options:
- (a) **Accept it:** run oracle over 375 with user-sim + judge on a cheap gpt-5.x; treat it as the gold/ceiling line (documented token cost, still << full agent run). Simplest, faithful.
- (b) **Scripted-user mode?** investigate whether tau2 supports a deterministic/replayed user (no user-sim LLM) for a truly-free gold — check `tau2 run` user options / a "solo"/scripted mode. If it exists, that's the 0-token gold.
- (c) **NL-judge determinism:** some tasks score purely on DB-state (`reward==1.0`) without NL assertions → those are LLM-free to score; split gold into DB-only (free) vs NL-assertion (judge-token) subsets.
**Recommend:** start with (a) for the gold ceiling; probe (b)/(c) to minimize token cost. Whatever we pick, the "oracle/gold 全量" DoD lane must state the token cost explicitly (it is not free).
